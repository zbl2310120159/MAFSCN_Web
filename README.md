## 项目简介

**MAFSCN 景区智能分析系统** 是一个基于 **MAFSCN**（Multi-Attention Fusion Sentiment Classification Network，多注意力融合情感分类网络）的深度学习情感分析与智能推荐系统。系统通过三个互补的注意力机制——通道注意力、空间注意力、交叉注意力——从 BERT 编码的文本特征中提取多维度情感语义，实现对景区游客评论的精准三分类（正面 / 中性 / 负面）。

在模型层面，系统采用 **双模型加权融合策略**：版本一模型在 6 个景区 2.3 万条评论上训练，对中性评论识别更为敏感；版本二模型覆盖 12 个景区 3.6 万条评论，在正负面判断上更加精准。两个模型按类别差异化加权融合后，整体情感分类准确率达到 **91.02%**，超越论文基准（84.58%）约 6.4 个百分点。

系统提供 **游客端界面** 和 **管理看板** 双界面，支持单条评论实时分析、景区智能推荐、批量处理和数据可视化，可广泛应用于景区口碑监控、游客满意度分析和舆情预警等场景。

**核心成果**：

- 📊 **情感分析准确率**：91.02%
- 🎯 **推荐系统 Top-5 准确率**：92.67%
- 🌍 **覆盖景区**：12 个大型旅游景区
- 🔗 **双模型融合**：按类别差异化加权，中性识别提升显著

## 模型架构

### MAFSCN 多注意力融合网络

MAFSCN 在 BERT 编码器之上构建了三路并行的注意力分支，从不同角度增强文本情感特征的表达能力：

| 注意力模块 | 作用对象 | 核心功能 | 情感分析价值 |
|-----------|---------|---------|-------------|
| **通道注意力** (Channel Attention) | CLS 向量 (768维) | 通过 FC→ReLU→FC→Sigmoid 学习维度权重，增强关键语义通道 | 识别与情感相关的语义维度（如"满意"vs"失望"的向量方向） |
| **空间注意力** (Spatial Attention) | 序列输出 (128×768) | 对每个 token 位置计算注意力分数，加权聚合 | 定位评论中情感关键位置（如"非常震撼"在句末的强调效果） |
| **交叉注意力** (Cross Attention) | 序列输出 (128×768) | 通过 QKV 自注意力捕捉 token 间长距离依赖 | 理解"虽然...但是..."等转折结构中情感的真正指向 |

三个分支的输出通过 **可学习的 Softmax 权重** 进行自适应融合：

```
融合特征 = w₁·通道输出 + w₂·空间输出 + w₃·交叉输出

其中 w₁, w₂, w₃ 由模型在训练中自动学习：
    训练后期 w₃（交叉注意力）增长至 ~0.59，成为主导分支
```

融合后的特征经 `Linear(768→256) → ReLU → Dropout(0.3) → Linear(256→3)` 输出三分类 logits。

### 双模型融合策略

两个模型在数据规模、景区覆盖和分类偏好上形成互补，通过 **按类别差异化加权** 实现融合：

| 模型版本 | 训练数据 | 数据量 | 核心优势 |
|----------|---------|--------|---------|
| **版本一** | 6 个景区 | 23,367 条 | 中性评论识别敏感，中性 F1 更高 |
| **版本二** | 12 个景区 | 36,000 条 | 正负面判断精准，覆盖景区更广 |

**融合权重设计：**

| 情感类别 | 版本一权重 | 版本二权重 | 设计思路 |
|----------|:--------:|:--------:|---------|
| 负面 | 30% | **70%** | 主要依靠版本二的正负面精准判断 |
| 中性 | **80%** | 20% | 主要依靠版本一的中性识别敏感性 |
| 正面 | 30% | **70%** | 主要依靠版本二的正负面精准判断 |

```
融合概率[负面] = 0.3 × P₁[负面] + 0.7 × P₂[负面]
融合概率[中性] = 0.8 × P₁[中性] + 0.2 × P₂[中性]
融合概率[正面] = 0.3 × P₁[正面] + 0.7 × P₂[正面]

融合后归一化，然后：
  if 融合概率[中性] ≥ 30% → 判定为"中性"
  else → 取融合概率[负面] 与 融合概率[正面] 中较大者
```

> **为什么中性阈值设为 30%？** 在常规三分类中，中性样本天然少于正负面，模型对中性的输出概率偏低。将阈值降至 30% 可有效提升中性召回率，同时不会误判高置信度的正负面样本。

### BERT 多分类推荐模型

```
用户输入文本
    ↓
BERT 编码
    ↓
[CLS] 特征向量 (768维)
    ↓
分类层 (12个景区)
    ↓
输出12个景区的概率分布
    ↓
Top-3 推荐排序 + 推荐理由
```

## 创新点

### 🌟 创新点一：双模型加权融合情感分析

**问题**：单一模型对中性评论识别率低，中性样本语义模糊且数量稀少（仅占 1.7%），传统三分类方法难以兼顾正负面精准度与中性召回率。

**解决方案**：训练两个差异化模型，按类别差异化加权融合
- **版本一**（6 景区 2.3 万条）：对中性评论识别敏感，中性 F1 更高
- **版本二**（12 景区 3.6 万条）：对正负面判断精准，覆盖景区更广
- **融合策略**：中性类别主要信赖 v1（0.8 权重），正/负类别主要信赖 v2（0.7 权重）

**效果**：
- 综合准确率达 91.02%，超越论文基准（84.58%）约 6.4 个百分点
- 中性识别能力显著提升，同时保持正负面高精准度

```
融合概率[负面] = 0.3 × P₁[负面] + 0.7 × P₂[负面]
融合概率[中性] = 0.8 × P₁[中性] + 0.2 × P₂[中性]
融合概率[正面] = 0.3 × P₁[正面] + 0.7 × P₂[正面]
```

### 🌟 创新点二：MAFSCN 多注意力融合网络

**问题**：单一注意力机制无法充分捕捉文本的多个维度特征，BERT 编码的文本特征仍有提升空间。

**解决方案**：融合三种不同类型的注意力机制
- **通道注意力**：识别哪些特征维度对分类最重要（如"满意"vs"失望"的向量方向）
- **空间注意力**：确定评论中哪些词汇位置信息量最大（如"非常震撼"在句末的强调效果）
- **交叉注意力**：在多个特征空间之间建立关联（如"虽然...但是..."等转折结构）

**效果**：
- 相比单一 BERT 编码提升准确率至 91.02%
- 三种注意力协同作用，充分利用文本信息
- 训练后期交叉注意力权重增长至 ~0.59，成为主导分支

```
BERT特征(768维)
    ↓ 分三路处理
通道注意力 + 空间注意力 + 交叉注意力
    ↓ 自适应融合（可学习 Softmax 权重）
融合特征(768维)
    ↓
分类输出
```

### 🌟 创新点三："推荐即分类"的思路

**问题**：景区推荐通常用协同过滤或内容相似度，对冷启动用户效果差，准确率有限。

**创新**：将推荐问题转化为多分类任务
- 将用户输入作为 BERT 输入
- 将 12 个景区作为分类标签
- 直接预测最可能的景区

**优势**：
- ✅ 充分利用 BERT 的文本理解能力
- ✅ 直接学习用户需求→景区的映射
- ✅ 准确率达到 92.67%（Top-5）
- ✅ 相比传统推荐算法简单高效，无需用户历史行为

### 🌟 创新点四：五维景区画像雷达图

**问题**：传统景区评价只看整体评分，无法直观展示各维度优劣势。

**解决方案**：从五个维度量化景区表现
- **风景**：自然景观、人文景观评分
- **服务**：导游、客服、管理水平评分
- **价格**：门票、餐饮、性价比评分
- **设施**：交通、卫生、休息区评分
- **体验**：整体游玩体验评分

**效果**：
- 雷达图直观展示各景区优劣势
- 支持多景区横向对比
- 辅助用户做出更明智的出行决策

### 🌟 创新点五：双界面系统设计

**问题**：单一界面无法同时满足游客和管理者的不同需求。

**解决方案**：
- **游客端**：情感分析 + 景区推荐 + 景区详情，操作简洁直观
- **管理看板**：数据统计 + 可视化图表 + 批量分析，深度数据洞察

**特点**：
- ✅ 游客端：3 秒内获取分析结果，推荐附理由和详情
- ✅ 管理端：ECharts 交互式图表，支持批量 CSV 导入分析
- ✅ 统一 Flask 后端，REST API 驱动，前后端解耦

## 常见问题

### 🔌 首次启动很慢？

需加载两个模型（~418MB×2）+ BERT 模型（~400MB），CPU 环境约需 10-30 秒，启动后所有请求复用。

### 📁 模型文件放哪里？

- 两个 `.pth` 文件必须放在 `results/models/` 目录
- BERT 模型必须放在 `./bert-base-chinese/` 目录
- 推荐模型 `bert_recommend_model.pth` 也放在 `results/models/` 目录

### 🖥️ Windows 编码问题？

启动 Web 服务时需设置 `PYTHONIOENCODING=utf-8`，或在命令行执行：
```bash
set PYTHONIOENCODING=utf-8
python app.py
```

### 📊 批量分析上限？

单次批量分析最多处理 5,000 条评论，超大文件请拆分后分批上传。

### 💾 没有 GPU 能用吗？

系统自动检测 CUDA 可用性，CPU 环境也可正常运行（推理速度稍慢，单条约 0.5-1 秒）。

### 🐍 Python 版本要求？

推荐 Python 3.10，最低支持 Python 3.8。

### 🔄 如何重新训练模型？

```bash
# 训练情感分析模型
python train.py sentiment

# 训练推荐模型
python train.py recommend

# 训练全部模型
python train.py all
```

训练参数可在 `config.py` 的 `TrainConfig` 中调整。
